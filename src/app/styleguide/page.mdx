### Unordered List

- The function to execute the mutation (e.g. mint a new NFT).
- `mutateAsync` - The function to execute the mutation, but returns a promise (allowing `await` to be used).
- `isLoading` - Whether the mutation is currently executing.
- `error` - The error returned from the mutation, if any.

### Ordered List

1. The function to execute the mutation (e.g. mint a new NFT).

3. `isLoading` - Whether the mutation is currently executing.

### Nested List

1. Nested lists are rarely a good idea

   - You might feel like you are being really "organized" or something but you are just creating a gross shape on the screen that is hard to read.
   - Nested navigation in UIs is a bad idea too, keep things as flat as possible.
   - Nesting tons of folders in your source code is also not helpful.
